# 提案：优化测试结果展示

## 为什么

当前测试历史页面显示的数据冗余且混乱，包含等级判断和重复的信息展示。用户无法快速获取最关键的数据，测试完成时的结算页面也存在同样问题。需要简化数据展示逻辑，让用户专注于最有价值的信息：个人最优成绩、百分位排名和完成日期。

## 改变什么

- **去除等级判断**：每种检测移除等级评分逻辑，专注展示客观数据
- **精简关键数据**：每种检测只显示自身的关键指标（如反应时间、准确率等）
- **优化历史页面**：
  - 每种测试仅显示最优结果
  - 显示超过 X% 的人（百分位排名）
  - 百分位 >99 时，显示小数点后两位
  - 显示完成日期
  - 点击项目查看详细信息
  - 支持清除成绩记录
- **统一结算页面**：与历史页面使用相同的展示规范
- **个人最优成绩提示**：检测完成后，若为个人最佳，显示恭喜提示

## 影响

- 影响规格：`test-result-display`、`test-history`
- 影响代码：
  - `src/pages/TestHistory/` - 历史记录页面
  - `src/pages/TestResult/` - 结算页面
  - `src/components/TestResultCard/` - 成绩卡片组件
  - `src/utils/score-calculation.ts` - 评分计算逻辑
  - `src/stores/useTestStore.ts` - 测试数据管理
